什么是Core Foundation

Apple官方文档对Core Foundation的定义。

Core Foundation is a library with a set of programming interfaces conceptually derived from the Objective-C-based Foundation framework but implemented in the C language. To do this, Core Foundation implements a limited object model in C. Core Foundation defines opaque types that encapsulate data and functions, hereafter referred to as “objects.”

Core Foundation是一个用C语言实现的编程接口库集合。从概念上来源于Obective-C的Foundation框架。Core Foundation用C实现了一个受限的对象模型。它定义了那些封装的数据类型和函数，也叫做对象。

Core Foundation在设计理念上是为了更方便的使用和复用，可以使代码在不同框架和库中使用。Core Foundation在某种程度上是跨平台的实现。具有支持国际化的unicode字符串，提供一些工具如插件，XML property lists URL资源访问及偏好设置。

CFAllocatorRef的结构

我们可以在CFBase.c中看到__CFAllocator的定义。

从第一句注释上看我们可以知道__CFAllocator的结构必须能与_malloc_zone_t匹配。在\usr\include\malloc\malloc.h中可以找到_malloc_zone_t的定义。

可以看到_malloc_zone_t与__CFAllocator不一样的地方就在前者的最后两个成员，和后者的最后一个成员。

根据注释，_malloc_zone_t的前两个成员就是为CFRuntimeBase占位的。最后的那个成员要在版本大于等于8才有，而现在分析的Core Foundation源码中version字段值都为6。

什么是CFRuntimeBase呢，我们可以在CFRuntime.h中找到它的定义。

uintptr_t是一个无符号的可以容纳指针的整形，也就是和void *是等价的。

uint8_t是1个字节，所以_cfinfo[4]占4个字节，当编译成64位程序的时候后面又加了一个4字节的成员uint32_t _rc，所以__CFRuntimeBase在32位程序下的大小是8字节，64位程序下是16字节。正好与_malloc_zone_t的前两个成员void *reserved1; void *reserved2;大小一致。

所有的CF对象实例都以CFRuntimeBase为起始，相当于都继承自CFRuntimeBase。CFRuntimeBase包含了isa指针，用来判断是否为CF对象，_cinfo[4]数组就是一个占了32位的空间用来保存bit field以此来保存CF对象信息。__CFAllocator大部分成员都为函数指针声明。这使得__CFAllocator像一个抽象类或者接口一样，可以把方法的实现放到定义的时候再决定。

分配内存

libc中有4种内存分配方式。malloc(), calloc(), realloc(), valloc()。

他们的区别在于calloc()把分配完的内存初始化为0。realloc()会增大分配的内存空间并返回一个新地址。valloc()分配一个按页大小对齐的内存。

CFAllocator的Public API，只有allocate与reallocate，但是也实现了calloc与valloc。

  • CFAllocatorAllocate

    如果allocator是NULL则获取一个默认的allocator。待分配的size为0返回空。如果allcator不是CF对象，那么按照malloc_zone_t分配内存。在GC环境下使用auto_zone_allocate_object()分配内存。如果是CF对象但不在GC环境那么使用allocator->context->allocate()函数中的方法分配内存。

    对于没有重新设置过默认allocator的话，那么使用的是__CFAllocatorSystemAllocate()。

  • CFAllocatorReallocate

    四种情况。

    • ptr == NULL && newSize > 0

      这种情况下的行为与malloc(newSize)相同。

    • ptr != NULL && newSize == 0

      将ptr指向的地址释放掉并返回NULL。

    • ptr == NULL && newSize == 0

      直接返回NULL。

    • ptr != NULL && newSize > 0

      调用allocator的realloc函数分配新的空间，如果分配成功的话，返回新分配的空间地址。

  • __CFAllocatorCustomCalloc

  • __CFAllocatorCustomValloc

    vm_page_size为一页的大小，值是4096字节，也就是4K字节。

    将CFAllocatorAllocate()分配出的内存地址加上vm_page_mask (= 4095)，再与vm_page_size的补码按位与得到一个新的地址。

释放内存

如果allocator是CFAllocator则调用allocator的dealloc方法否则调用malloc_zone_free()。

对于默认的Allocator，调用的是free()函数。

像__LP64__这种宏定义在编译器中的预处理器中。可以在Predefined Macros查看预处理器定义了哪些宏。

总结

CFBase最主要的就是CFAllocatorRef了，CFAllocatorRef封装了系统的内存分配与释放，使其可以跨平台使用。用户也可自定义内存分配与释放方法。
